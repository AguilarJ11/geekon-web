# GeekOn! — Roadmap de Perfil de Usuario

## Contexto
GeekOn es la web de la comunidad geek de Uruguay. Ya existe una página de
perfil de usuario (`/perfil` y `/perfil/editar`) con los siguientes campos
y secciones implementadas:

- Avatar y banner
- Nombre
- Bio (280 caracteres)
- Badge de rol "Miembro"
- Fecha de ingreso ("Miembro desde...")
- Sección "Logros & Badges" (UI lista, sin lógica de badges todavía)
- Accesos para postularse a: Artista, Stand Comercial, Concurso Cosplay,
  Actividad

## Objetivo
Ir extendiendo el perfil de usuario con nuevas funcionalidades, implementadas
de a una por vez (no todas juntas), en el orden de prioridad definido abajo.
Cada feature debe quedar funcionando y probada antes de pasar a la siguiente.

## Orden de implementación

### Fase 1 — Datos básicos del perfil (baja complejidad)
- [x] Campos de redes sociales (Instagram, Discord, TikTok, X/Twitter)
- [x] Campo de ciudad/zona (Uruguay)
- [x] Tags de intereses / fandoms favoritos (anime, cómics, gaming, cosplay,
      sci-fi, etc.), mostrados como chips en el perfil

### Fase 2 — Panel de postulaciones
- [x] Vista "Mis postulaciones" que muestre el estado (pendiente / aprobada /
      rechazada) de las postulaciones enviadas
- [x] Conectar esta vista con los formularios/backend ya existentes

Nota (2026-07-23): se decidió NO hardcodear categorías fijas (Artista, Stand,
Cosplay, etc.) en el código. El admin crea cada postulación como un
formulario real desde /admin/formularios (form builder ya existente) con
el nombre que quiera (Stand Comercial, Concurso de Cosplay, Torneo de
TCG, Torneo de Videojuegos, Pasarela de Cosplay...), y recién ahí queda
disponible para que los usuarios se postulen. /perfil lista automáticamente
los formularios publicados a los que el usuario no se postuló, y su estado
una vez enviada. El admin aprueba/rechaza desde
/admin/formularios/[id]/respuestas.

### Fase 3 — Gamificación e identidad
- [x] Rol visible según participación (Cosplayer, Artista, Staff, Visitante)
- [x] Sistema de niveles / puntos de experiencia
- [x] Logros específicos conectados a la sección "Logros & Badges" ya existente

Nota (2026-07-23): un usuario puede tener más de un rol visible a la vez
(staff + participante + organizador), se muestran todos como fila de
badges, no solo el de mayor prioridad.

Se agregó el concepto de "dueño de formulario": el admin puede asignarle
a cualquier usuario (por email) la propiedad de un formulario puntual
(ej. "Torneo de Tekken" → Abstract Gaming). Ese dueño administra
únicamente su propio formulario desde /mis-formularios (campos,
publicar/despublicar, aprobar/rechazar postulaciones), sin acceso al
resto del panel admin. Al asignarse como dueño recibe automáticamente
un badge de organizador ligado a la categoría del formulario (ej.
"Expositor / Taller", "Organizador de Torneo").

Cada formulario tiene además un campo "edición" (texto libre, ej.
"GeekOn 2026"). Las medallas por postulación aprobada son genéricas
por categoría + edición (no por formulario específico): cualquier
torneo de esa edición, sin importar la temática, otorga la misma
medalla "Competidor — GeekOn 2026". Se agregó también un estado
"ganador" (además de pendiente/aprobada/rechazada), marcable solo
sobre postulaciones aprobadas, que otorga una medalla extra tipo
"Ganador de Cosplay — GeekOn 2026".

Nivel y XP se calculan al vuelo (no se persisten) a partir de
postulaciones aprobadas y badges obtenidos: no hay tabla de puntos.

Pendiente para más adelante (fuera de esta fase): logros basados en
asistencia a ediciones pasadas del evento ("asistió a 3 ediciones")
requeriría un sistema de check-in/asistencia que todavía no existe.

### Fase 4 — Social y pulido
- [ ] Seguir / agregar a otros usuarios de la comunidad
- [ ] Mini portfolio o galería personal (para artistas y cosplayers)
- [ ] Configuración de privacidad del perfil (público / solo comunidad)
- [ ] Historial de ediciones de GeekOn en las que participó

## Notas de trabajo
- Cada fase se desarrolla feature por feature, no en bloque.
- Antes de arrancar una fase nueva, confirmar que la anterior está probada
  en producción.
- Este documento se actualiza a medida que se completan features (marcar
  checkboxes) o se agregan nuevas ideas.

## Ideas a futuro (sin prioridad definida)

- Modo de edición visual para administradores: permitir arrastrar y
  reposicionar assets/elementos de la página en tiempo real (tipo editor
  de diseño). Requiere definir antes: qué elementos son editables, si los
  cambios se ven en vivo para todos los visitantes o se aplican al
  recargar, y qué mecanismo de rollback existe si algo se rompe. Evaluar
  cuando las fases 1 a 4 del perfil estén completas.
